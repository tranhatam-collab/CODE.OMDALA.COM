# CODE OF CONDUCT

OMCODE Community Code of Conduct

OMCODE is an open-source project built for serious collaboration.

We want this project to be:
	•	respectful
	•	technically rigorous
	•	practical
	•	inclusive
	•	focused on useful work

This Code of Conduct applies to:
	•	repository discussions
	•	issues
	•	pull requests
	•	review comments
	•	documentation discussions
	•	community interactions connected to the project

⸻

1. Our Standards

Behavior we expect
	•	Be respectful
	•	Be constructive
	•	Be honest
	•	Be specific
	•	Be open to feedback
	•	Assume good intent, but focus on real impact
	•	Keep discussions grounded in the project and the work
	•	Help make the project more usable and more trustworthy

Behavior we do not accept
	•	personal attacks
	•	harassment
	•	insults
	•	discrimination
	•	intimidation
	•	trolling
	•	doxxing
	•	repeated bad-faith arguments
	•	hostile or degrading language
	•	deliberately disruptive behavior

⸻

2. Technical Disagreement Is Allowed

Strong technical disagreement is normal.

It is acceptable to:
	•	question an architecture choice
	•	request evidence
	•	point out risks
	•	reject weak proposals
	•	ask for clearer scope
	•	challenge a design decision

It is not acceptable to:
	•	attack the person instead of the work
	•	mock contributors
	•	belittle less experienced developers
	•	derail discussions with hostility

⸻

3. Communication Style

We encourage communication that is:
	•	direct
	•	clear
	•	useful
	•	respectful

Good examples:
	•	“This approach may break the route contract because…”
	•	“This adds scope beyond the current milestone.”
	•	“Please add tests for this behavior.”
	•	“This needs to align with the Mac alpha priority.”

Bad examples:
	•	“This is stupid.”
	•	“You clearly do not understand anything.”
	•	“Why are you even here?”
	•	“This is trash.”

⸻

4. Scope and Focus

This project has a defined direction.

Community members should respect:
	•	the current roadmap
	•	the current architecture
	•	the current product priorities
	•	the project maintainers’ final decisions

Not every idea belongs in the project immediately.

⸻

5. Inclusive Collaboration

We welcome contributors from different backgrounds, levels of experience, and regions.

We do not tolerate discrimination based on:
	•	race
	•	ethnicity
	•	nationality
	•	language background
	•	religion
	•	disability
	•	gender
	•	sexual orientation
	•	age

⸻

6. Maintainer Responsibilities

Project maintainers are responsible for:
	•	enforcing this code of conduct
	•	keeping discussions productive
	•	protecting contributors from abuse
	•	making fair moderation decisions
	•	prioritizing project health over unnecessary conflict

Maintainers may:
	•	ask for tone changes
	•	close derailing discussions
	•	remove abusive comments
	•	lock threads
	•	reject or remove harmful contributors when necessary

⸻

7. Reporting Problems

If you experience or witness unacceptable behavior:
	•	contact the maintainers through the designated project contact path
	•	include relevant links, screenshots, or context if possible
	•	do not escalate publicly if private reporting is safer

Sensitive reports should be handled with care and discretion.

⸻

8. Enforcement

Possible enforcement actions include:
	•	warning
	•	comment removal
	•	temporary interaction limits
	•	issue or PR lock
	•	contribution suspension
	•	permanent ban from project spaces

The response should be based on severity, pattern, and impact.

⸻

9. Good Faith

People make mistakes.

A respectful correction, apology, and changed behavior may be enough in many cases.

Repeated harmful behavior is different and may require stronger action.

⸻

10. Project Culture

OMCODE is being built as a serious product and open-source system.

That means we value:
	•	clarity over noise
	•	substance over ego
	•	collaboration over posturing
	•	maintainability over chaos
	•	trust over hype

⸻

11. Final Note

By participating in this project, you agree to follow this Code of Conduct.

Help us build a project that is not only technically strong, but also healthy to work in.
